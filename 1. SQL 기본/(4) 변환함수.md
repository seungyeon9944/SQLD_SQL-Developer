- **암시적 형변환** (DB가 내부적으로 알아서 데이터유형 변환)
- **명시적 형변환** (변환 함수 사용 → 데이터 유형 변환 명시적으로 나타냄)

### `TO_NUMBER`

문자를 숫자형으로 변환

✏️ 오답노트

```sql
SELECT EMPNO
FROM EMP
ORDER BY TO_CHAR(TO_NUMBER(SUBSTR(COL1, 3, 2))); # 1, 11, 12, 6, 8
```

### `TO_CHAR(수 or 날짜 [,포맷])`

포맷 형식의 문자형으로 변환

```sql
SELECT TO_CHAR(1234) FROM DUAL; # '1234'
SELECT TO_CHAR(SYSDATE, 'YYYYMMDD') FROM DUAL; # '20251110'
```

✏️ 오답노트 :
*문자값의 대소비교는 값의 왼쪽부터 하나씩 !*

```sql
SELECT SAL
FROM TAB1
ORDER BY TO_CHAR(SAL); # 1300 2500 300 800 8000
```

### `TO_DATE(문자열, 포맷)`

포맷형식의 문자열을 날짜형으로 변환

```sql
SELECT TO_DATE('20251110', 'YYYYMMDD') FROM DUAL; # 2025-11-10
```

*✏️ -30/24/60은 30 * 1/24/60인데 **1/24/60이 1분**이므로 기준시간에서 30분 빼면 됨*
